# Immortal Architecture – 受講者ガイド

このフォルダは教材を購入してくれた方向けのハブです。「アンチパターン API」を触りながら、なぜ設計が崩れるのかを体験してもらうための読み物とハンズオン手順をまとめています。以下の順番で読み進めると迷いません。

1. **anti_pattern_design.md**  … ゴールとアンチパターンの全体像。「何を観察すればいいか」をざっくり掴むためのイントロダクションです。
2. **anti_pattern_walkthrough.md** … 実際のコード片を見ながら「どこでどんな痛みが発生しているか」を確認します。各節に“Try it”メモがあるので、手元で再現しながら読み進めてください。
3. **anti_pattern_fix.md** … 同じ問題をどう直すかのアイデア集。アンチパターンを見た直後に「じゃあどうする？」が分かるように噛み砕いてあります。
4. **migrations.md** … DB 初期化や差分反映の手順。手元で API を動かしたいときに参照してください。

> 📌 **おすすめの進め方**
> 1. `anti_pattern_design.md` を読みながら API のエッセンスとアンチパターンの一覧を把握する。
> 2. VS Code や IDE でコードを実際に開きつつ `anti_pattern_walkthrough.md` を読み、該当ファイルを自分の目で確認する。
> 3. 「何が辛いか」が腑に落ちたら、`anti_pattern_fix.md` を読み、現場ではどう直していくかのヒントを得る。
> 4. 必要に応じて `migrations.md` を参考にローカル DB を用意し、実際に API を叩いて挙動を確かめる。

この 4 ファイルで教材の世界観が完結します。不要なファイルは整理済みなので、最新情報は常にここから辿れるようになっています。EOF
